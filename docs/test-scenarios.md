# 수동 테스트 시나리오

이 세션(2026-07-21~22)에 고친/추가한 기능을 직접 눌러보면서 확인하기 위한 체크리스트다. 운영 배포
(https://tour-dna.lib.lc) 또는 로컬(`npm run dev` → http://localhost:3000)에서 위에서부터 순서대로
진행하면 된다. 각 항목은 "무엇을 하고 → 무엇이 보여야 정상인지"로 적었다.

## 0. 사이트 접근 게이트 (SITE_ACCESS_PASSWORD를 설정한 경우에만 해당)

- [ ] 시크릿/무개인정보 창으로 아무 페이지(`/`, `/projects/new` 등)에 접속 → `/login`으로 자동 이동하는지
      확인
- [ ] 틀린 비밀번호 입력 → "비밀번호가 올바르지 않습니다." 오류가 뜨고 페이지에 그대로 남는지 확인
- [ ] 맞는 비밀번호 입력 → 원래 가려던 페이지로 이동하는지 확인
- [ ] 상단 메뉴의 "잠금" 버튼 클릭 → `/login`으로 돌아가고, 다시 아무 페이지나 들어가려 하면 로그인을
      또 요구하는지 확인
- [ ] (설정 안 했다면) 위 항목은 전부 건너뛰고 그냥 바로 들어가지는지 확인 — 게이트가 없으면 이게 정상

## 1. 새 프로젝트 생성 화면 (`/projects/new`)

- [ ] 시/도에서 **대전광역시** 선택 → 시/군/구 드롭다운에 "대전광역시"가 또 나오지 않고
      **"대전광역시 (DNA 지표는 유성구 기준)"** 하나만 나오는지 확인(예전엔 시/도명이 그대로 중복 표시되던
      버그였음)
- [ ] 시/도에서 **강원특별자치도** 선택 → 시/군/구에 **강릉시/양양군**이 나오는지 확인(2026-07-21에
      추가된 지역)
- [ ] 시/도를 바꾸면 프로젝트명 제안이 즉시 바뀌는지 확인(예: "대전광역시 9월 소규모 여행 기획" →
      "강원특별자치도 9월 소규모 여행 기획"), 프로젝트명을 직접 타이핑하면 그 이후로는 자동 제안이
      끊기고 입력한 값이 유지되는지 확인
- [ ] "분석 시작" 버튼에 마우스를 올리면 커서가 손가락(포인터) 모양으로 바뀌는지 확인(전체 페이지의
      다른 버튼들 — 실행안의 위/아래/삭제, 인쇄, 오류 화면 "다시 시도" 등도 동일하게 확인 가능)

## 2. 관광 DNA 분석 화면 (`/projects/[id]/analysis`)

- [ ] 대전광역시로 프로젝트를 만들어 분석 실행 → 수요/체류/소비/다양성/연계 5축 점수가 전부 정확히
      0점 또는 100점만 나오지는 않는지 확인(2026-07-21에 비교 지역을 3개→7개로 늘려 완화한 부분 —
      완전히 없어지지는 않으므로 일부는 여전히 0/100일 수 있음, "완화"이지 "해결"은 아님)
- [ ] "근거 보기"를 눌러 각 지표의 원값/정규화값/단위/기준월/출처/반영 규칙이 보이는지 확인
- [ ] 전략 3안이 서로 다른 점수/순위/근거 문구를 갖는지 확인

## 3. 전략 선택 → 실행안 (`/projects/[id]/plan`)

- [ ] 1순위가 아닌 다른 전략을 선택 → 상품명/콘셉트 문구/코스가 실제로 바뀌는지 확인(예전엔 전략을
      바꿔도 실행안이 그대로였던 버그였음)
- [ ] 코스의 장소들이 지리적으로 말이 되는 순서로 배치되는지 확인(예: 같은 지역 안에서 왔다갔다 하지
      않고 자연스럽게 이어지는지) — 이동 텍스트에 "이동 약 N분(약 X.Xkm, OO 기준)" 형태로 실제 거리
      기반 추정치가 나오는지 확인

### 3-1. 코스 편집 — 순서 변경

- [ ] 장소 옆 ↑/↓ 버튼으로 순서를 바꿔본다 → 이동 텍스트(travel)는 새로 이웃하게 된 장소 쌍 기준으로
      다시 계산되지만, **각 장소가 가진 시간(왼쪽의 시간 입력창)은 그대로 유지**되는지 확인(순서와
      시간이 분리되어 있음 — 예전에는 시간이 자리에 묶여 있었다)

### 3-2. 코스 편집 — 시간 직접 수정

- [ ] 아무 장소의 시간 입력창(⏰)을 클릭해 다른 시각으로 바꿔본다 → 바로 반영되는지 확인
- [ ] 일부러 **시간이 거꾸로 가게** 만들어본다(예: 뒤 순서 장소의 시간을 앞 장소보다 이르게 설정) →
      해당 장소 칸이 **빨간 테두리/배경**으로 바뀌고 "⚠ 이전 일정이 OO에 끝나는데 이 장소는 그 전에
      시작합니다" 같은 경고 문구가 뜨는지 확인
- [ ] 일부러 **이동시간보다 여유를 짧게** 만들어본다(예: 두 장소 시간 차이를 5분으로 줄이는데 실제
      필요한 이동시간은 그보다 김) → 마찬가지로 빨간 경고가 뜨는지 확인
- [ ] 시간을 다시 여유 있게 고치면 경고가 사라지는지 확인

### 3-2b. 코스 편집 — 체류시간 직접 수정

- [ ] 체류시간 숫자 입력(기본 60분)을 다른 값으로 바꿔본다 → 바로 반영되는지, 값을 늘려서 다음 장소와의
      여유가 부족해지면 위 3-2의 빨간 경고가 뜨는지 확인

### 3-3. 코스 편집 — 삭제

- [ ] 아무 장소나 "삭제" 버튼 클릭 → 목록에서 바로 빠지는지 확인

### 3-4. 코스 편집 — 다른 날짜로 이동 (1박 2일 이상 프로젝트에서만 가능)

- [ ] 장소 옆 "N일차" 선택창에서 다른 날짜를 고른다 → 원래 날짜에서는 빠지고 고른 날짜에 나타나는지,
      **원래 갖고 있던 시간을 그대로 유지**하는지 확인(옮긴 날짜에 이미 비슷한 시간대 장소가 있으면
      위 3-2의 빨간 경고가 뜰 수 있음 — 정상 동작)

### 3-5. 코스 편집 — 장소 검색 후 추가

- [ ] 날짜 아래 "+ 장소 추가" 클릭 → 검색창이 열리는지 확인
- [ ] 장소 이름 일부를 입력 → 같은 지역에 등록된 장소 중 일치하는 것만 결과로 나오는지 확인(이미 코스에
      들어있는 장소는 검색 결과에서 자동으로 빠짐)
- [ ] 결과에서 "추가" 클릭 → 해당 날짜 맨 끝에 새 장소가 추가되고, 검색창이 자동으로 닫히는지 확인
- [ ] **한 날짜에 4곳을 이미 다 채운 상태에서도** "+ 장소 추가"가 계속 활성화되어 있고, 5번째·6번째도
      제한 없이 추가되는지 확인(예전엔 "이 날짜는 가득 찼습니다"로 막혀 있었음)

### 3-6. 운영 체크리스트/위험과 대응안/KPI 편집

- [ ] "운영 체크리스트" 아래 입력창에 텍스트를 넣고 "추가" 클릭 → 목록에 바로 추가되는지 확인, 항목별
      "삭제" 버튼으로 지울 수 있는지 확인(위험과 대응안·KPI도 각각 동일하게 추가/삭제 가능한지 확인)
- [ ] KPI 메모/메모 입력란은 그대로 있는지 확인(이번에 손대지 않은 부분)
- [ ] 저장 후 인쇄 화면(`/print`)에서 방금 추가/삭제한 체크리스트·위험·KPI 내용이 개수 제한 없이 전부
      반영돼 보이는지 확인(예전엔 체크리스트 5개까지만, 위험과 대응안은 아예 안 보였음)

### 3-7. 저장/이탈

- [ ] 상품명이나 코스를 수정하면 화면 오른쪽에 "저장하지 않은 변경사항이 있습니다"가 뜨는지 확인
- [ ] "저장" 클릭 → "모든 변경사항이 저장되었습니다"로 바뀌는지, 새로고침(F5) 후에도 수정한 내용이
      그대로 남아있는지 확인
- [ ] 수정 후 저장하지 않고 브라우저 탭을 닫거나 새로고침하려 하면 브라우저 자체의 "변경사항이
      저장되지 않을 수 있습니다" 경고가 뜨는지 확인(단, 앱 내부 링크로 이동할 때는 이 경고가 뜨지
      않는 게 정상 — 알려진 제한사항)

## 4. 인쇄/PDF (`/projects/[id]/print`)

- [ ] 실행안 화면에서 "인쇄/PDF 보기" 클릭 → 화면 조작 버튼(위/아래/삭제/저장 등) 없이 깔끔한 인쇄용
      레이아웃만 보이는지 확인
- [ ] 화면 상단의 "← 실행안으로 돌아가기" 링크로 이전 화면으로 돌아갈 수 있는지 확인(예전엔 이 화면에
      진입하면 되돌아갈 방법이 없었음)
- [ ] 브라우저 인쇄 미리보기(Ctrl+P) 시 A4 1~2페이지 분량으로 나오는지 확인

## 5. 지도 (`/projects/[id]/analysis` 하단)

- [ ] 카카오맵 키가 정상 등록되어 있으면 실제 지도가, 없으면 좌표/주소 목록 fallback이 뜨는지 확인

### 5-1. 실행안 코스 지도 (`/projects/[id]/plan`, 2026-07-22 추가)

- [ ] "일자·시간대별 코스" 위쪽에 지도가 뜨고, 코스 장소들이 방문 순서대로 마커+선(동선)으로 이어지는지
      확인
- [ ] 1박 2일 이상 프로젝트에서는 "1일차/2일차" 탭이 뜨고, 탭을 누르면 해당 날짜 동선만 다시 그려지는지
      확인(1일짜리 프로젝트는 탭 없이 바로 지도만 보임)
- [ ] 마커를 클릭하면 "N. 시간 장소명" 정보창이 뜨는지 확인
- [ ] 코스에서 장소를 추가/삭제/다른 날짜로 이동하면 지도도 그에 맞춰 다시 그려지는지 확인
- [ ] 좌표 정보가 없는(예전에 저장된) 실행안이라면 "좌표 정보가 있는 장소가 없어 동선을 표시할 수
      없습니다" 안내가 뜨는지 확인(오류가 아니라 정상적인 안내임)
- [ ] 아직 지도에서 장소를 검색해 코스에 바로 추가하는 기능은 없다 — 검색은 여전히 3-5의 텍스트
      검색창에서만 가능함(알려진 제한사항)
