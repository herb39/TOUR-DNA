# 실제 경로 API(Phase 12) 무료 사용 조건 확인 현황 — REVIEW_ONLY, 2026-07-23

마스터 프롬프트 12-1절: "코드를 수정하기 전에 2026년 7월 현재 공식 문서와 카카오디벨로퍼스 콘솔에서
다음을 확인"하라는 조건이 걸려 있다. 이 문서는 **코드 검토 결과**만 담는다 — 카카오디벨로퍼스 콘솔
확인은 Claude Code가 대신 로그인해서 할 수 없으므로, 전부 사용자 확인 대기(`BLOCKED`)로 남긴다.

## 1. 현재 코드 상태 (확인 완료)

- `src/components/map/CourseMap.tsx`, `src/components/map/kakaoLoader.ts`는 **카카오맵 JavaScript SDK**만
  사용한다(`Marker`/`InfoWindow`/`Polyline`/`LatLngBounds`). 좌표 사이를 직선으로 잇는 Polyline이며 실제
  도로 경로가 아니다.
- 카카오내비 길찾기(`apis-navi.kakaomobility.com`), 카카오맵 REST 경로 조회 API 호출 코드는 저장소
  전체에 존재하지 않는다(검색 결과 0건: `KAKAO_NAVI`, `RouteProvider`, `directions`).
- 서버 전용 REST API 키(`TOUR_API_SERVICE_KEY`와 별개의 카카오 REST 키)는 `.env.example`에 없다 —
  현재는 `NEXT_PUBLIC_KAKAO_MAP_KEY`(브라우저 노출용 JS 키) 하나만 존재.
- 즉 Phase 12는 **코드베이스 관점에서 0%, 전혀 착수되지 않았다.**

## 2. 사용자가 카카오디벨로퍼스 콘솔에서 직접 확인해야 할 항목 (BLOCKED)

확인할 수 없다(로그인/콘솔 접근 필요, 그리고 결제 관련 설정은 안전 규칙상 대행 금지 대상이기도 하다).

- [ ] 자동차 길찾기 무료 일간 쿼터 10,000건 / 다중경유지 길찾기 5,000건 — 현재 앱에 실제 적용되는지

- [x] **2026-07-21부터 카카오맵 무료 쿼터가 "개발자 계정에서 처음 카카오맵 API를 활성화한 앱"에만
      적용된다는 조건** — 2026-07-22 사용자가 콘솔에서 직접 확인, TOUR DNA 앱에 "카카오맵 무료 쿼터"

      보관기간 제한 등)

## 2-1. 사용자 확인 결과 (2026-07-22)

- **확인일**: 2026-07-22
- **확인 내용**: 카카오디벨로퍼스 콘솔의 TOUR DNA 앱에 **"카카오맵 무료 쿼터" 배지가 표시됨**을 사용자가
  직접 콘솔에서 확인함 — 위 2절 세 번째 항목("2026-07-21부터 무료 쿼터는 개발자 계정에서 처음 카카오맵
  API를 활성화한 앱에만 적용"되는 조건의 대상 앱임)이 이 배지로 충족된 것으로 확인됨.
- **비즈월렛/유료 API**: 비즈월렛은 연결하지 않았고, 유료 API도 활성화하지 않았음(사용자 확인).
- **API 키 값**: 실제 키 값은 이 문서와 코드 어디에도 기록하지 않는다(3-1절 원칙 그대로 유지) — 콘솔
  확인 결과(배지 표시 여부)만 기록한다.
- **남은 미확인 항목**: 자동차/대중교통/도보 각 엔드포인트의 실제 무료 일간 쿼터 수치, 서버 전용 REST
  API 키 발급 여부와 도메인/IP 화이트리스트, 길찾기 API 신청 상태, 응답 저장·재게시 관련 이용약관
  제약 — 이 4가지는 여전히 사용자 확인 대기 상태이며, 위 2절 체크리스트에서 미해결로 남아 있다.

## 3. 결론

- **현재 시점(2026-07-23): 일부 확인됨, 전체 착수 조건은 아직 미충족.** 2026-07-22 확인으로 "최초
  활성화 앱 무료쿼터 배지"와 "비즈월렛 미연결/유료 API 미활성화" 2개 항목은 해소되었으나, 실제 일간
  쿼터 수치·서버 REST 키 발급·이용약관 제약 등 나머지 항목이 남아 있어 Phase 12는 여전히 `BLOCKED`다.
- 마스터 문서 12-1절의 명시적 지시대로, 비즈월렛 연결·유료 API 활성화·카드 등록·종량제 계약이 필요한
  것으로 확인되면 이 Phase는 시작하지 않는다.
- 위 체크리스트를 사용자가 콘솔에서 확인해 답변을 주기 전까지 Phase 12는 `docs/implementation-status.md`
  기준 `BLOCKED`로 유지한다. 확인 후에도 무료 쿼터 초과 시 자동 유료 전환이 아니라 429 폴백(Haversine
  직선거리 추정)으로 처리하는 것이 마스터 문서 12-4절의 요구사항이며, 이는 확인 완료 후 구현 단계에서
  지켜야 할 제약으로 이 문서에 기록해 둔다.
- 무료 조건이 충족되지 않아도 **기존 Haversine 기반 일정/지도는 그대로 정상 동작해야 한다**(12-5절) —
  이는 이미 현재 구현 상태이므로 Phase 12를 시작하지 않아도 서비스에는 영향이 없다.
